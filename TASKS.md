## 🔧 Backend

### 1. Инфраструктура и DevOps

- [ ] 🔴 **Dockerfile и docker-compose** — создать `Dockerfile` для backend, `docker-compose.yml` с PostgreSQL + Redis + backend + frontend
- [ ] 🔴 **CI/CD pipeline** — GitHub Actions: линтинг, тесты, сборка образа, деплой
- [ ] 🔴 **Нормальные Alembic миграции** — переписать `initial_schema` на реальные `op.create_table()` вместо `create_all()`. Без этого невозможно делать инкрементальные миграции
- [ ] 🔴 **Убрать `.env` из репозитория** — добавить в `.gitignore`, создать `.env.example` с шаблоном
- [ ] 🔴 **Создать `.gitignore`** — `.env`, `__pycache__`, `.venv`, `node_modules`, `*.db`, `dist/`, `.idea/`, `*.log`
- [ ] 🟡 **Логирование** — настроить structlog или стандартный logging с JSON-форматом, ротацией, уровнями по модулям
- [ ] 🟡 **Health check расширить** — добавить проверку БД и Redis
- [ ] 🟡 **Rate limiting** — slowapi или middleware для защиты от брутфорса (особенно /auth/login)
- [ ] 🟡 **CORS** — вынести `allowed_origins` в `.env`, убрать `*` для methods/headers в production
- [ ] 🟢 **Мониторинг** — Sentry для ошибок, Prometheus-метрики
- [ ] 🟢 **API versioning** — вынести `/api/v1` в конфиг для будущих v2

### 2. Аутентификация и безопасность

- [ ] 🔴 **Refresh token с автообновлением** — фронт должен вызывать `/auth/refresh` при 401, сейчас просто чистит localStorage
- [ ] 🔴 **Password validation** — добавить проверку сложности пароля (минимум 8 символов, цифры, буквы)
- [ ] 🟡 **Email verification** — подтверждение email при регистрации
- [ ] 🟡 **Password reset** — забыл пароль → email → сброс
- [ ] 🟡 **Device tracking** — поле `device_id` в RefreshToken есть, но не заполняется
- [ ] 🟡 **Login throttling** — блокировка после N неудачных попыток входа
- [ ] 🟡 **Superadmin seed** — скрипт для создания первого суперадмина
- [ ] 🟢 **OAuth2** — вход через Google/Telegram (опционально)
- [ ] 🟢 **2FA** — двухфакторная аутентификация для owner/admin
